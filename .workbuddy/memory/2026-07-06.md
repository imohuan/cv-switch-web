## 根因：全局 settings.json 覆盖 Profile 配置
- **根因**: Claude Code 启动时 merge 全局 `~/.claude/settings.json` 和 Profile 的 `HOME/.claude/settings.json`，全局的 `env` 块优先级更高。全局配置中残留了 `ANTHROPIC_BASE_URL: http://newapi.imohuan.shop/v1` 等旧值，导致 Claude Code 直接请求 New API 而非走本地 Proxy。
- **修复**: 清理全局 `~/.claude/settings.json`，移除所有 `ANTHROPIC_*` 环境变量，仅保留 `skipDangerousModePermissionPrompt` 和 `theme`。
- **后端** `backend/src/index.ts`: 请求日志中间件从仅记录 `/proxy/` 改为全量记录所有请求（含 host、user-agent、状态码、耗时）。
- **后端** `backend/src/services/providerConfig.ts`: `publicBaseUrl()` 默认端口从 `3001` 改为 `3120`，与 `index.ts` 一致。
- **后端** `backend/src/services/profiles.ts` + `claude.ts`: ANTHROPIC_MODEL 等模型名改为 `claude-sonnet-4-20250514`，避免 Claude Code 客户端白名单校验拦截。实际模型映射由 Proxy 内部用 `provider.model` 完成。
- **后端** `backend/src/services/profiles.ts` + `claude.ts`: ANTHROPIC_API_KEY 从 `"PROXY_MANAGED"` 改为使用 **provider 真实 API key**。Claude Code 的 `~/.claude.json` 中有已批准的 key 列表（`customApiKeyResponses.approbed`），只有已批准的 key 才能绕过 `/login` 检查。之前用假 key 导致 Claude Code 判定"未登录"。
- **根因**: Claude Code 启动时会检查当前 `ANTHROPIC_API_KEY` 是否在已批准列表中（通过 key 尾缀匹配）。不在列表中 → 显示 "Not logged in · Please run /login"，**不会发出任何 HTTP 请求**到 BASE_URL。
- **修复方案**: Profile 配置直接写入 Provider 的真实 API key（与全局已批准的 key 一致），Proxy 仍然用数据库中的 key 发请求给上游。
- **重要补充**: Claude Code 的认证状态（`numStartups`、`customApiKeyResponses`、`projects`）存储在 `~/.claude.json`（**注意不在** `.claude/` 子目录内）。当用户用 `$env:HOME` 切换到 Profile 目录后，Claude Code 在新 HOME 下找不到这个文件也会判定 "Not logged in"，**同样不会发任何 HTTP 请求**。
- **修复**: 后端 `backend/src/services/profiles.ts` 新增 `copyClaudeAuthState()` 函数，在写 Claude Profile 时自动从全局 `~/.claude.json` 复制到 Profile 的 `~/.claude.json`。以后创建新 Profile 不再需要手动操作。
- **Windows 上 HOME 无效**: Claude Code 在 Windows 上不读取 `HOME` 环境变量，而是使用 `USERPROFILE`（通过 `os.homedir()`）。`CLAUDE_CONFIG_DIR` 虽然可以覆盖配置目录，但会导致找不到凭证。最终方案：**不用 HOME 也不用 CLAUDE_CONFIG_DIR**，直接在命令行设置 `ANTHROPIC_BASE_URL` / `ANTHROPIC_API_KEY` / `ANTHROPIC_MODEL` 等环境变量。Claude Code 使用全局凭证认证，API 请求走 Proxy。
- **Express body parser 限制**: 默认 100kb，Claude Code 请求含大量 system prompt + tools 定义，请求体可达数 MB。改为 `express.json({ limit: '50mb' })`。
- **HEAD 探测请求**: Claude Code 启动时发 HEAD 请求到 BASE_URL 探测服务器存活。Proxy 需要处理 HEAD 请求返回 200，否则 Claude Code 判定连接失败。

## 模型角色映射（Haiku/Sonnet/Opus）
- **问题**: 之前所有环境变量都设为同一个 `claude-sonnet-4-20250514`，Proxy 用 `provider.model` 发请求，无法区分角色。Provider 配置的 Haiku/Sonnet/Opus 模型完全被忽略。
- **修复**: 不同角色用不同的 Claude 模型名（都在白名单内），Proxy 根据名称映射到 Provider 实际模型：
  - `ANTHROPIC_MODEL` / `ANTHROPIC_DEFAULT_SONNET_MODEL` → `claude-sonnet-4-20250514` → Proxy 映射到 `extra_config.claude.sonnetModel`
  - `ANTHROPIC_DEFAULT_HAIKU_MODEL` / `ANTHROPIC_SMALL_FAST_MODEL` → `claude-haiku-4-5-20251001` → Proxy 映射到 `extra_config.claude.haikuModel`
  - `ANTHROPIC_DEFAULT_OPUS_MODEL` → `claude-opus-4-8-20250610` → Proxy 映射到 `extra_config.claude.opusModel`
- **涉及文件**: `backend/src/routes/claudeProxy.ts` 新增 `mapClaudeModelToProvider()` 函数；`backend/src/services/profiles.ts` 和 `backend/src/services/claude.ts` 更新模型名设置。
