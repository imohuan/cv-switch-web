# 2026-07-06

## 多平台启动命令功能
- **后端** `backend/src/services/profiles.ts`: 新增 `launchCommands()` 函数，返回 `{bash, powershell, cmd}` 三种平台格式。原 `launchCommand()` 保留作为向后兼容（返回 bash 格式）。新增 `LaunchCommands` 和 `CommandPlatform` 类型导出。
- **后端** `backend/src/routes/providers.ts`: `profileResponse()` 和 GET `/api/profiles` 接口新增 `commands` 字段。
- **前端** `frontend/src/api.ts`: Profile 接口新增 `commands: {bash, powershell, cmd}` 字段。
- **前端** `frontend/src/App.vue`: 启动命令区域从单行代码改为带 Tab 切换的多平台面板，支持 Bash/PowerShell/CMD 三种格式切换和复制。新增 `profileCommandPlatform` 状态管理每个 Profile 的选中平台。

## Provider 选择后自动回填模型
- **前端** `frontend/src/App.vue`: 新增 `watch(profileProviderId, ...)` 监听，切换 Provider 时自动从 `extra_config.{claude,codex}` 回填默认模型、Haiku/Sonnet/Opus 模型、Codex 模型目录到表单字段。
- 之前创建 Profile 时 `extra_config` 为 `{"claude":{}}`（空），因为表单字段从未自动填充。修复后选择 Provider 即可看到预配置的模型值。
## 修复 Claude Profile 双重 Auth 冲突
- **后端** `backend/src/services/claude.ts`（全局配置）: 移除 `settings.env.ANTHROPIC_AUTH_TOKEN` 写入，改用 `delete` 清理旧值。
- **后端** `backend/src/services/profiles.ts` 的 `writeClaudeProfile()`（Profile 配置）: 同样移除 `ANTHROPIC_AUTH_TOKEN`。之前只修了全局的，遗漏了 Profile 专用的写入函数，导致「重新写入配置」仍然输出 AUTH_TOKEN。
## Claude 二进制自动复制
- **后端** `backend/src/services/profiles.ts`: 新增 `copyClaudeBinary()` 函数，写 Claude Profile 时自动从全局 `~/.local/bin/claude.exe` 复制到 Profile 的 `.local/bin/claude.exe`，避免每个 Profile 都需要手动跑 `claude install`。
## 后台日志系统
- **后端** `backend/src/services/logger.ts`: 统一日志模块，控制台 + 文件双输出（路径: `~/.cc-switch-web/logs/cc-switch-YYYY-MM-DD.log`），按天轮转。提供 `claudeProxy` / `codexProxy` / `models` / `profiles` 四个子模块标签。
- **后端** `backend/src/routes/claudeProxy.ts`: `handleClaudeMessages` 集成日志——记录请求开始、上游请求 URL/Model、响应成功（含延迟 ms）或失败（含上游错误 body 预览前 500 字符）。
- **后端** `backend/src/routes/codexProxy.ts`: `handleCodexResponses` 同样集成日志。
- **后端** `backend/src/routes/providers.ts`: 模型列表获取端点集成日志；新增 GET `/api/logs` 端点返回最近 200 行日志（供前端查看）。
- **后端** `backend/src/routes/providers.ts`: 修复 `require('fs')` / `require('path')` 为 ESM `import`。
